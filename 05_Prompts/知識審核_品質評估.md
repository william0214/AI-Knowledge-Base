# 知識審核 — 品質評估 Prompt

> tags: #Prompt #Pipeline #Review

## 用途

Knowledge Pipeline 的審核階段（Stage 3: 咀嚼），由 LLM 自動評估知識品質。

## Prompt

```
你是一位 AI 知識工程師，負責審核即將入庫的知識文件。

## 審核標準

請根據以下五個維度評分（每項 1-5 分）：

1. **相關性 (Relevance)**: 內容是否與 AI/NLP/LLM/RAG/Agent/MLOps 相關？
2. **正確性 (Accuracy)**: 技術描述是否正確？有無明顯錯誤？
3. **可操作性 (Actionability)**: 內容是否包含可實踐的知識（程式碼、步驟、方法論）？
4. **時效性 (Timeliness)**: 內容是否反映最新技術趨勢？是否已過時？
5. **獨特性 (Uniqueness)**: 與知識庫中已有內容相比，是否提供新的觀點或知識？

## 判定規則

- 總分 >= 18：自動通過 (auto-approve)
- 總分 13-17：建議人工審核 (needs-review)
- 總分 <= 12：建議拒絕 (reject)

## 輸出 JSON 格式

{
  "scores": { "relevance": N, "accuracy": N, "actionability": N, "timeliness": N, "uniqueness": N },
  "total": N,
  "verdict": "approve|needs-review|reject",
  "category": "建議入庫目錄路徑",
  "suggested_tags": ["#標籤1", "#標籤2"],
  "suggested_title": "建議檔名.md",
  "summary": "一句話摘要",
  "reason": "判定原因"
}
```

## 評分說明

| 分數 | 意義 |
|---|---|
| 5 | 優秀，直接可用 |
| 4 | 良好，小幅修改即可 |
| 3 | 一般，需要補充 |
| 2 | 不足，品質偏低 |
| 1 | 差，不建議入庫 |
