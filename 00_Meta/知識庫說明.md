# 知識庫說明

> tags: #Meta

## 目的

本知識庫是**個人知識管理系統（Personal Knowledge Management）**，最終目標是成為**個人 AI 機器人的記憶來源**。

知識庫涵蓋範圍：
- 📚 **外訓課程**：課堂筆記、實作紀錄、外訓心得
- � **技術自學**：YouTube 影片、線上課程、技術文章
- 💼 **工作專案**：專案經驗、技術決策、踩坑紀錄
- 🔧 **個人 SOP**：工具備忘、環境設定、工作流程
- 🌐 **產業觀察**：趨勢分析、研討會心得、技術觀點

所有知識最終會被提煉、結構化，匯入 `08_Bot_Knowledge/`，作為個人 AI 機器人的可檢索知識庫。
### Knowledge Pipeline 知識管線

知識入庫採用四階段流水線：

| 階段 | 名稱 | 說明 | 位置 |
|---|---|---|---|
| ① | 爬 (Crawl) | 爬蟲自動抓取 YouTube / 網頁 | `09_Staging/raw/` |
| ② | 消化 (Digest) | LLM 自動摘要、結構化、分類 | `09_Staging/digested/` |
| ③ | 哀嚎 (Review) | LLM 評分 + 人工審核把關 | `09_Staging/reviewed/` |
| ④ | 產出 (Output) | 自動分類入庫 + 更新索引 | 正式目錄 + `08_Bot_Knowledge/` |

詳見 [Knowledge Pipeline 架構設計](../03_Architecture/Knowledge_Pipeline.md)  
操作手冊 [Knowledge Pipeline SOP](../12_SOP/Workflows/Knowledge_Pipeline_操作手冊.md)

## 使用方式

### 外訓課程筆記流程

1. **上課時**：在 `01_Courses/課程名稱/` 對應章節中記錄筆記
2. **課後整理**：將可複用的概念整理到 `02_Concepts/` 中
3. **實作練習**：記錄在 `04_Experiments/`
4. **完課後**：彙整為外訓心得

### 日常知識累積流程

1. **學習**：在 `10_Learning/` 記錄 YouTube、課程、文章筆記
2. **工作專案**：在 `11_Projects/專案名稱/` 記錄技術決策與經驗
3. **寫 SOP**：在 `12_SOP/` 記錄可重複使用的流程與工具設定
4. **產業觀察**：在 `13_Insights/` 記錄趨勢與觀點

### 知識歸檔流程（手動）

```
原始筆記 → 概念提取（02_Concepts/）→ 架構設計（03_Architecture/）→ Bot 知識（08_Bot_Knowledge/）
```

### Knowledge Pipeline流程（自動）

```
爬蟲抓取 → 09_Staging/raw/ → LLM 消化 → 09_Staging/digested/ → 人工審核 → 09_Staging/reviewed/ → 入庫
```

## 知識庫維護

### 每日習慣（15-30 分鐘）

- 整理當日學習筆記或工作紀錄
- 將零散筆記拆解為概念卡（`02_Concepts/`）
- 重要的架構概念繪製圖表放入 `07_Diagrams/`
- 實用的 Prompt 範例收集到 `05_Prompts/`

### 每週回顧

- 檢視 `#todo` 和 `#in-progress` 標籤的檔案
- 將已完成的筆記標記為 `#done`
- 考慮是否有新的概念卡或 SOP 需要撰寫

### 每月整理

- 將過時的內容移到 `99_Archive/`
- 更新 `08_Bot_Knowledge/` 的彙整知識
- 回顧 `13_Insights/` 調整技術方向
