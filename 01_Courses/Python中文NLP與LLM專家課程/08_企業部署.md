## 8.7 圖表補述：Flask RAG 與 vLLM 多 GPU 部署圖

### 💡 思考
- 這幾張圖真正的重點，是「如何部署」，還是「如何把多個元件拆開」？
- 在部署圖中，哪一條箭頭應被看作 API 請求，哪一條是容器內部控制？

### 📌 重點
- Flask RAG 圖可還原為：
  - `HTTP Request` -> `Flask app` -> `Retriever / Vector Store` -> `LLM` -> `JSON Response`
- UUUGPT 設定圖可還原為：
  - `下載壓縮檔` -> `解壓到指定資料夾` -> `修改 config.py` -> `python app.py`
- vLLM 多 GPU 圖可還原為：
  - `client 容器` -> `vLLM OpenAI-compatible API` -> `4 張 GPU tensor parallel` -> `生成結果`
  - 旁路還有 `health check` 與 `restart on-failure`，代表服務治理而不是推理主流程。
- 壓測圖可還原為：
  - `stress_vllm.sh` -> `併發請求` -> `vLLM API` -> `耗時 / RPS 指標`

### 🔧 實作

# 來源：課程講義 PDF 第 57-59, 112-116 頁
