## 1.1 課程定位與主軸

### 💡 思考
- 這門課的重點是中文 NLP、LLM 應用，還是企業部署？
- 各模組之間的依賴順序是什麼？
- 若要在企業做 PoC，應該先落哪一段？

### 📌 重點
- 課程不是單點工具教學，而是一條完整的企業 AI 落地路徑。
- 內容主軸分為三層：
  - 中文 NLP 基礎：爬蟲、分詞、Embedding、文本分類、情感分析。
  - LLM 應用：Transformer、RAG、Assistants API、LangChain、LlamaIndex、Agent。
  - 企業部署：微調、GGUF、LM Studio、vLLM、多 GPU 推理與私有化部署。
- 課程編排的實質邏輯是：先建立資料與 baseline，再導入 LLM 強化與部署。

## 1.3 課程學習地圖

### 💡 思考
- 這門課的真正閱讀順序，是照檔名走，還是按概念依賴走？
- 哪些章節是基礎，哪些章節是建立在前面成果上的延伸？

### 📌 重點
- 01–08 可以看成五個階段：
	- `01-02`：課程定位與 NLP / ML 基礎觀念。
	- `03-04`：資料蒐集、前處理、中文分詞與向量化。
	- `05`：文本分類與情感分析任務化。
	- `06-07`：RAG、Agent、微調與模型能力強化。
	- `08`：API、LM Studio、vLLM 與 MLOps 部署。
- 依賴關係上，`03` 和 `02` 共同支撐 `04`，`04` 又進一步支撐 `05` 與 `06`。
- 若只想快速進入企業落地視角，建議先看 `01 -> 06 -> 07 -> 08`，再回補 `02-05` 的技術細節。
